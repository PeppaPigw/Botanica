# Security Policy

Thank you for helping keep Botanica and its users safe.

## Reporting a vulnerability

**Preferred:** please report security issues using **GitHub Security
Advisories** so we can coordinate a private fix and disclosure.

- Go to the repository Security tab and choose “Report a vulnerability”, or use
  the advisory creation link if available.

**Fallback:** if you cannot use a private advisory, you may open a GitHub issue
with a minimal description and ask maintainers to move the report to a private
channel. Please **do not** include exploit code, credentials, or other
sensitive details in a public issue.

## Scope

We welcome reports for security issues in the code and workflows contained in
this repository, including:

- The Flutter mobile app (Android / iOS)
- Build scripts and project tooling
- GitHub Actions workflows
- The optional proxy/server code shipped in `server/` (as code-in-repo)

Out of scope (examples):

- Issues in third‑party services or dependencies that do not affect this repo’s
  code directly
- Problems in **your** deployment configuration, infrastructure, or accounts
- Social engineering, physical attacks, or device compromise

## Coordinated disclosure

Please allow reasonable time to investigate and patch before public
disclosure. If you have a deadline, include it in your report.
